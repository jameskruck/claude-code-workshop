Create a Course Outline

Strategic planning • 10 minutes

📋 Your Task

You're working with an SME who has rough notes for a B2B SaaS pricing course. Transform these messy notes into a professional, QM-compliant course outline.

Step 1: Have Claude Read the File

The file is already in your directory. Ask Claude to read it:

Read the file sample-course-brief.md

Claude will use the Read tool to access the file and show you the contents.

Step 2: Invoke the Agent

Type this prompt (or copy from below):

Use the course-outline-creator agent to transform this course brief into a complete 5-week MBA course outline. Include: CLOs using QM standards, weekly structure with MLOs, assessment strategy with alignment matrix, concept threading map, and practitioner integration plan. Ask me about course format (cohort vs self-paced) and any other details you need.

Step 3: Answer Questions

The agent will ask about:

  • Course format: Say "cohort-based" (fixed dates, allows peer review)
  • Duration: Confirm "5 weeks"
  • Other clarifications: Answer based on the brief content or your judgment

✅ What You'll Get

  • 5 Course Learning Outcomes (CLOs) with single action verbs, Bloom's levels
  • Weekly structure with themes, MLOs (4-5 per week), case studies, practitioners
  • Assessment strategy with alignment matrix showing which assessments cover which CLOs
  • Concept threading map showing how Week 1 concepts build across Weeks 2-5
  • Practitioner integration plan with specific roles and topics

💡 Pro Tip: Notice how the agent references the Ivey 6-phase development process and concept threading patterns automatically - that's the bundled knowledge base in action!

2

Build Week 1 Storyboard

Detailed module design • 15 minutes

📋 Your Task

Take the course outline you just created and generate a detailed Week 1 storyboard with copy-paste-ready content for implementation in your LMS.

Key insight: This demonstrates how agents work together in a pipeline (strategic planning → detailed design)

Step 1: Use Your Outline

You already have the outline from Exercise 1 in your Claude Code session - no need to upload anything new!

Step 2: Invoke the Storyboard Builder

Type this prompt:

Use the uplimit-storyboard-builder agent to create a detailed Week 1 storyboard based on the course outline we just created. I need copy-paste-ready content for all modules including: element tables, text blocks written in full, infoboxes, widget specifications, video scripts, assessment instructions, and design rationale for each choice. Use the varied content delivery principles to ensure interactive, engaging design.

Step 3: Review the Output

The storyboard builder will create a comprehensive implementation guide with:

  • Module-by-module breakdown (typically 7-8 modules for Week 1)
  • Element tables showing element type, duration, priority
  • Complete text blocks - ready to copy into your LMS
  • Widget specifications with interaction flows and success states
  • Infoboxes with full content and icons
  • Assessment instructions with rubrics
  • Design rationale explaining pedagogical choices

🔍 What to Look For

  • Interactivity: Notice how it balances text, video, widgets (30/70 passive/active ratio)
  • Threading: Week 1 introduces all core concepts that will be built upon in Weeks 2-5
  • Knowledge checks: Formative assessments every 3-5 minutes of content
  • Varied delivery: Long text blocks broken into multiple short elements with different formats

💡 Pro Tip: Ask follow-up questions! "Can you add more interactive widgets to Module 3?" or "Expand the assessment section with more detailed rubric criteria."

Choose Your Next Tool

Based on your immediate needs • 5-10 minutes

🎯 Your Choice

Pick the exercise that matches your current work priorities. Both demonstrate different plugin capabilities you'll use regularly.

Option A: Generate Assessment Rubric

Best if you: Need to create grading criteria for assessments

Steps:
  1. Use the storyboard's assessment from Exercise 2 (already in your session)
  2. Type: Use the rubric-generator agent to create a detailed rubric for the Week 1 assessment in the storyboard
  3. Review the 100-point rubric created
  4. Try asking: "Add a student-facing version without point breakdowns"

You'll Get: QM-aligned rubric with 4 performance levels (Exemplary/Proficient/Developing/Needs Improvement), evaluation criteria aligned to learning objectives, point distribution, student & faculty versions

Or Try This Prompt:
Use the rubric-generator agent to create a rubric for a pricing strategy case analysis assignment. Students analyze a company's pricing evolution and recommend changes. Create a 100-point rubric with clear evaluation criteria for: strategic thinking, analytical rigor, practical recommendations, and business writing quality. Include 4 performance levels.

Option B: Audit Accessibility

Best if you: Want to check content for WCAG compliance

Steps:
  1. Ask Claude: Create a simple HTML page with an h1 heading, a paragraph, an image, and a button
  2. Then type: Use the accessibility-auditor agent to check this HTML for WCAG 2.2 AA compliance
  3. Review the violations found (likely missing alt text, button label issues)
  4. Try asking: "Fix all violations"

You'll Get: Line-by-line identification of: low contrast text, missing alt text, improper heading hierarchy, missing button labels, semantic HTML issues, color contrast violations

Or Use Your Own HTML:
If you have HTML content from a real project, you can ask Claude to read that file and use the accessibility-auditor agent to check it for WCAG 2.2 AA compliance. The agent will identify all violations with specific line numbers and provide copy-paste fixes.

💡 Have Time for Both? Great! Try both exercises to see the full range of what the plugin can do. Each takes only 5-10 minutes.
