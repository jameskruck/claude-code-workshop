Activity 1: Setup & Deploy Workshop

Deploy the workshop with your own AI assistant, then set up additional accounts

üí°

Workshop Setup Note

During the live session: We'll demonstrate using the instructor's deployed version with working AI chat.

For hands-on practice: You'll create your own copy with your own API key - this ensures you control your usage and costs.

Your deployed workshop will be completely independent and private to you!

1

Fork & Deploy Workshop

Get your own copy of this workshop running online with AI chat capabilities.

Open Workshop Repository

What to do:

  • Click "Fork" in the top-right corner
  • Create GitHub account if needed (use work email)
  • Keep default settings and click "Create fork"
  • You now have your own copy!
2

Deploy to Render

Get your workshop running online so you can test the AI chat functionality.

Open Render & Sign Up

What to do:

  • Sign up with GitHub (same email as your fork)
  • Authorize Render to access your repositories
  • Click "New +" ‚Üí "Web Service"
  • Select your forked "claude-code-workshop" repo
  • Keep all default settings and click "Create Web Service"
  • Wait 2-3 minutes for deployment to complete
3

Add Your AI Assistant

Enable the AI chat by adding your Anthropic API key to your deployed workshop.

Get Anthropic API Key

Step 3.1: Get your API key

  • Sign up/login at console.anthropic.com
  • Go to Settings ‚Üí API Keys
  • Click "Create Key" and copy it

Step 3.2: Add to Render

  • Go back to your Render dashboard
  • Click on your deployed workshop service
  • Go to "Environment" tab
  • Add: Key = ANTHROPIC_API_KEY, Value = your copied key
  • Click "Save Changes" (app will redeploy automatically)

Activity 3: Build Your Tool

Use Claude Code to create your educational tool from start to finish

1

Access Claude Code

Open claude.ai/code in a new tab

2

Copy Your Starting Prompt

Feedback Collector Prompt:
Create a student feedback collection web application. Include: a form with name, course, 5-star rating for overall satisfaction, 5-star rating for content quality, text area for comments, and a submit button. Also create a simple results page that shows all submitted feedback in a clean list. Use modern, clean styling with a professional look. Make it mobile-friendly.
Study Timer Prompt:
Create a Pomodoro study timer web application. Include: 25-minute work timer, 5-minute break timer, start/pause/reset buttons, session counter, and a simple progress display. Add notification sounds when timers complete. Use clean, focused design that helps concentration. Make the timer large and easy to read.
Quiz Builder Prompt:
Create a simple quiz application. Include: a way to add multiple-choice questions with 4 options each, a quiz-taking interface that shows one question at a time, immediate feedback on answers, final score display, and ability to retake the quiz. Use engaging colors and clear typography. Make it intuitive for both creating and taking quizzes.
Learning Journal Prompt:
Create a daily learning journal web application. Include: date picker, reflection prompts (What did I learn today? What challenged me? What will I do differently?), text areas for responses, save entry button, and a view to see past entries by date. Use calming colors and readable fonts that encourage reflection.
3

Paste & Send

Paste the prompt into Claude Code and hit enter to start building!

Activity 4: Deploy Your Tool

Get your educational tool online and share it with others

1

Create New Repository

Save your educational tool code to GitHub for deployment

Ask Claude Code:
"Help me create a new GitHub repository for this project and push all my files. I want to deploy this educational tool online. Walk me through each step."

Claude Code will help you:

  • Create a new GitHub repository
  • Initialize git in your project
  • Add and commit your files
  • Push everything to GitHub
  • Prepare for deployment
2

Deploy to Render

Make your educational tool accessible online for your students and colleagues

In Render Dashboard:
  1. Click "New +" ‚Üí "Web Service"
  2. Connect your new GitHub repository
  3. Render auto-detects the app type
  4. Click "Create Web Service"
  5. Wait 2-3 minutes for deployment
  6. Get your live URL and test your tool!
Need help with deployment issues?
"My deployment failed with this error: [paste error]. Can you help me fix this and get my educational tool online?"

üí° Building Tips

  • Start with the basic functionality first
  • Test your tool frequently as you build
  • Ask Claude Code to explain any code you don't understand
  • Request styling improvements after core features work
  • Add one feature at a time

üîÑ Common Follow-up Requests

Styling: "Make the design more modern and add some color"
Features: "Add a way to delete entries" or "Include data validation"
Layout: "Make this mobile-friendly" or "Center the content better"
Fixes: "The submit button isn't working" or "The timer resets too early"
