Activity 3: Build Your Tool

Use Claude Code to create your educational tool from start to finish

1

Access Claude Code

Open claude.ai/code in a new tab

2

Copy Your Starting Prompt

Feedback Collector Prompt:
Create a student feedback collection web application. Include: a form with name, course, 5-star rating for overall satisfaction, 5-star rating for content quality, text area for comments, and a submit button. Also create a simple results page that shows all submitted feedback in a clean list. Use modern, clean styling with a professional look. Make it mobile-friendly.
Study Timer Prompt:
Create a Pomodoro study timer web application. Include: 25-minute work timer, 5-minute break timer, start/pause/reset buttons, session counter, and a simple progress display. Add notification sounds when timers complete. Use clean, focused design that helps concentration. Make the timer large and easy to read.
Quiz Builder Prompt:
Create a simple quiz application. Include: a way to add multiple-choice questions with 4 options each, a quiz-taking interface that shows one question at a time, immediate feedback on answers, final score display, and ability to retake the quiz. Use engaging colors and clear typography. Make it intuitive for both creating and taking quizzes.
Learning Journal Prompt:
Create a daily learning journal web application. Include: date picker, reflection prompts (What did I learn today? What challenged me? What will I do differently?), text areas for responses, save entry button, and a view to see past entries by date. Use calming colors and readable fonts that encourage reflection.
3

Paste & Send

Paste the prompt into Claude Code and hit enter to start building!

15 min

Activity 4: Deploy & Share

Get your tool online using GitHub and Render

1

Save to GitHub

Store your code in GitHub for version control and deployment

Ask Claude Code:
"Help me save this to GitHub. I need to create a new repository and push my code. Guide me through each step."

Claude Code will help you:

  • Create a new GitHub repository
  • Initialize git in your project
  • Add and commit your files
  • Push everything to GitHub
2

Deploy with Render

Make your tool accessible online with free hosting

In Render Dashboard:
  1. Click "New +" → "Web Service"
  2. Connect your GitHub repository
  3. Render auto-detects it's a Node.js app
  4. Click "Create Web Service"
  5. Wait 2-3 minutes for deployment
  6. Get your live URL!
If something goes wrong:
"My Render deployment failed. Here's the error message: [paste error]. Can you help me fix this?"
3

💡 Building Tips

  • Start with the basic functionality first
  • Test your tool frequently as you build
  • Ask Claude Code to explain any code you don't understand
  • Request styling improvements after core features work
  • Add one feature at a time

🔄 Common Follow-up Requests

Styling: "Make the design more modern and add some color"
Features: "Add a way to delete entries" or "Include data validation"
Layout: "Make this mobile-friendly" or "Center the content better"
Fixes: "The submit button isn't working" or "The timer resets too early"
